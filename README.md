# Swim Meet Checker (web app)

Live at **https://yutingf.github.io/swimmeet/** (a standalone project site, separate from
the academic homepage at yutingf.github.io and never linked from it).

A self-contained website that shows Ethan and Lucas's current times and, when you drop or
paste a PVS meet-announcement PDF, tells you what each boy qualifies for and which events
to sign up for, with the meet event number on each. Everything runs in your browser: the
PDF is parsed locally with pdf.js, and the times are baked into `swim-data.js`. No server
logic, no account, no AI, no tokens.

## Keeping the published site's times current
The site shows whatever is in `swim-data.js` at publish time. After the boys race, refresh
and re-publish: run `code/swim_data.py` (regenerates `webapp/swim-data.js`), then push the
updated `webapp/` contents to the `yutingf/swimmeet` repo. The times only change after
meets, so this is occasional.

## Daily use (fully private, on this Mac)

Double-click **`Open swim website.command`** in the `swimming-team` folder. It refreshes
the boys' times (USA Swimming API plus NVSL), serves this folder at
`http://localhost:8777`, and opens it. Drag a meet PDF onto the page. Close the Terminal
window when done.

A local server is used rather than opening `index.html` directly because Chrome blocks
pdf.js's worker under `file://`; the launcher serves over `http://localhost`, where it
works.

## Files
- `index.html`, `style.css`: the page.
- `app.js`: pure logic (PDF grid parsing, eligibility, value ranking). Mirrors the
  Python engine in `../code/`; both are covered by `../code/test_webapp.js`.
- `ui.js`: reads the DOM, runs pdf.js, paints results.
- `swim-data.js`: the boys' times plus motivational standards, regenerated by
  `../code/swim_data.py` on every refresh. This is the only file that changes over time.
- `vendor/pdf.min.js`, `vendor/pdf.worker.min.js`: pdf.js, vendored so nothing loads
  from the internet.

## Publishing to GitHub Pages (optional)
Copy this `webapp/` folder into a repo and enable Pages (Settings, then Pages, deploy from
the folder). The site is then reachable at `https://<user>.github.io/<repo>/`.

Privacy note: on a personal GitHub account, a Pages site is public to anyone who has the
URL even when the source repo is private (access-controlled Pages needs GitHub
Enterprise). The data shown (names, times) is already public record on USA Swimming and
NVSL. For fully private use, use the local launcher above instead of Pages. Refresh times
by running `code/swim_data.py` and committing the updated `swim-data.js`.
